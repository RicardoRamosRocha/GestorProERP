# Veltis ERP — Planejamento das Sprints

## Objetivo

Este documento define as sprints de desenvolvimento do Veltis ERP.

Cada sprint representa um conjunto de funcionalidades que serão implementadas e entregues de forma incremental.

---

# Sprint 1 — Fundação da Solução

## Objetivos

* Criar solução ASP.NET Core
* Configurar arquitetura em camadas
* Configurar SQL Server
* Configurar Entity Framework Core
* Configurar GitHub
* Criar banco de dados inicial

## Entregáveis

* Estrutura da solução criada
* Banco configurado
* Primeira migration criada

# Critérios de Conclusão

Uma sprint será considerada concluída quando:

* Funcionalidades implementadas
* Testes realizados
* Código versionado
* Documentação atualizada
* Commit realizado no GitHub

# Visão Geral do Projeto

Planejamento → Arquitetura → Desenvolvimento → Testes → Publicação

O desenvolvimento seguirá entregas incrementais para garantir qualidade, organização e evolução contínua do sistema.
